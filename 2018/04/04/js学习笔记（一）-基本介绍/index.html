js学习笔记（一）- 基础

发表于 2018-04-04 | 分类于 javascript | 阅读次数

javascript历史

一开始提出javascript只是为了改变原有的表单输入验证功能（发回到服务器再做验证），而目前javascript已经发展成为一门具备与浏览器进行全面交互的一门强大语言。

javascript组成部分

  • ECMAScript : 由ECMA-262定义，提供核心语言功能。
  • 文档对象模型（DOM）：提供访问和操作网页内容的方法和接口。
  • 浏览器对象模型（BOM）: 提供与浏览器交互的方法和接口(弹出新浏览器窗口、移动缩放关闭浏览器窗口、对cookies的支持…)。

DOM级别

  • DOM1级由两个模块组成：DOM核心和DOM HTML。DOM核心规定如何映射基于XML的文档结构，以简化对文档任意部分的访问和操作。DOM HTML模块则在DOM核心的基础上加以扩展，添加了针对HTML的对象和方法。
  • DOM2级进一步扩充鼠标和用户界面事件、范围、遍历（迭代DOM文档的方法）等细分模块，CSS支持…
  • DOM3级: DOM Load and Save、DOM validation.

<script>元素

属性

  1. async（异步脚本）:可选，表示立即下载脚本但不应妨碍页面中的其他操作，比如下载其他资源或等待加载其他脚本，只适用于外部脚本。告诉浏览器立即下载文件但页面不会等待脚本的下载和执行，而是异步加载页面其他内容。

    标记为async的脚本并不保证按照它们的前后顺序执行。异步脚本一定在页面load事件前执行，但可能在DOMContentLoaded事件触发之前或之后执行。支持异步脚本浏览器有Firefox3.6、Safari 5和Chrome。

    在XHTML文档中，async属性设置：async=”async”.

  2. charset :可选。指定src代码的字符集。（大多数浏览器会忽略）

  3. defer（延迟脚本）: 可选。脚本延迟到文档完全被解析和显示之后再执行。只对外部脚本文件有效。告诉浏览器立即下载但延迟执行。

    HTML5规范要求脚本按照它们出现的先后顺序执行，即先执行1.js再执行2.js，而这两个脚本会先于DOMContentLoaded事件执行。而现在实际上，延迟脚本并不一定会按照顺序执行，也不保证会在DOMContentLoaded事件前执行。为了保证兼容性，把延迟脚本放在页面底部仍然是最佳选择。

  4. language:已废弃。

  5. src: 引入外部文件。

  6. type：可选。默认值就是text/javascript。

    所有<script>元素都会按照它们在页面中出现的先后顺序依次被解析（除去使用async或defer的情况）
    一般把<script>元素放在页面后面，</body>之前。
